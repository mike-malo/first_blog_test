用户定义的函数的使用方式和内置函数完全相同，我们无法从sum_square的定义中判断square是否是用户定义的、模块导入的还是内置在解释器的

环境（environment）：

计算表达式的环境由一系列执行帧组成，每个执行帧包含绑定，每个绑定都将一个名称和其相应的值相关联，函数也出现在环境图中。import语句将名称绑定到内置函数。def语句将名称绑定到由定义创建的用户定义函数

每个函数都是一行，以func开头，后跟函数名和形式参数。像mul这样的内置函数没有正式的参数名，所以它总是被…用来代替。

函数名重复两次，一次在帧中，另一次作为函数本身的一部分。函数中出现的名称称为内在名称（intrinsic name）。框架中的名称是绑定名称（bound name）。两者之间有一个区别：不同的名称可能指的是同一个函数，但该函数本身只有一个固有名称。

函数签名（Function Signatures）：

函数允许接受的参数数量不同。为了追踪这些需求，我们以显示函数名及其形式参数的方式绘制每个函数。用户定义的函数square只需要x；提供更多或更少的参数将导致错误。函数形式参数的描述称为函数的签名。

函数max可以接受任意数量的参数。它被渲染为max（…）。

无论需要多少参数，所有内置函数都将呈现为（…），因为这些原始函数从未明确定义过。

调用用户定义函数(调用用户定义函数):

为了计算其运算符命名用户定义函数的调用表达式，Python解释器遵循一个计算过程。与任何调用表达式一样，解释器计算运算符和操作数表达式，然后将命名函数应用于结果参数。

Review

除法（Division）

/ (True Division): 结果为浮点数，例如：1 / 5 结果为0.2, 25 / 4 结果为6.25

// (Floor Division): 结果为四舍五入（round down）的整数，例如：1 // 5 结果为0, 25 // 4结果为6，注意：四舍五入取最小，例如：-25 // 4 结果为 -7

% （remainder）: 求余数，涉及到一个重要点：检查某数是否能够被另一个数整除

函数（Functions）

如果我们要一遍又一遍地执行一系列语句，我们可以将它们抽象成一个函数以避免重复代码。

调用函数

在调用函数过程中，可能需要也可能不需要参数，每个呼叫表达式都需要一组括号来划定其逗号分隔的操作数。

计算函数调用：

  • 计算操作符，然后计算操作数（从左到右）。

  • 将操作符应用于操作数（操作数的值）。

如果操作数是嵌套的调用表达式，则将这两个步骤应用于该内部操作数，以评估外部操作数。

return 和 print

您定义的大多数功能都包含返回语句。返回语句将使一些计算的结果回到函数的呼叫者并退出函数。例如，下面的函数square取用数字x并返回其平方。

当Python执行返回语句时，该函数会立即终止。如果Python在不执行返回语句的情况下到达函数体系的末端，则它将自动返回无。相反，打印函数用于在终端显示值。这可能会导致打印和返回之间的混淆，因为在Python解释器中调用功能将打印出函数的返回值。但是，与返回语句不同，当Python评估打印表达式时，该函数不会立即终止。

控制（Control）

布尔c（Boolean Operators）

python 提供三个布尔操作符and, or, not

布尔v，例如算术v，都有一个计算顺序：

  • not有最高优先级

  • and

  • or优先级最低

事实证明 and 和 or 计算不仅仅是布尔值（True，False）。 Python值，例如 0，none，**’’**（空字符串）和 []（空列表）被视为False。所有其他值都被认为是真实值。

Python的 and 和 or 操作符短路。也就是说，他们不一定会评估每个操作数。
